# 中文模型映射说明

这个仓库不强制你使用某一套固定模型映射，但建议你遵守一个原则：

**用户看到的模型名要稳定，实际映射的上游可以在后台演进。**

## 推荐做法

- 暴露少量、稳定、可理解的模型名
- 文档里明确说明每个模型名大致对应什么能力
- 不要把大量内部 alias 直接暴露给用户

## 示例思路

- `claude-sonnet-4-6`：默认均衡模型
- `claude-opus-4-6`：更强推理模型
- `claude-haiku-4-5`：更快更便宜的模型

重点不在于“这三个一定怎么映射”，而在于：

- 用户端名字尽量长期稳定
- 你能在后台替换真实上游
- 文档能解释清楚速度、效果、价格差异
