# 中文限制说明

## 请求兼容，不等于环境兼容

这个项目可以帮你做：

- 请求格式兼容
- 配置收敛
- 客户端接入简化

但它不能替你做：

- 本地 Git 仓库准备
- 本地 `git worktree` 创建
- Claude Code 的子代理隔离环境

## 为什么这一点很重要

如果 Claude Code 只是正常单线程工作，这类 bridge 很有用。

但如果 Claude Code 本地决定要起 subagent 或 swarm 风格的多 agent：

- 仍然需要本地 Git/worktree 环境
- 仍然可能受本地仓库状态影响

## 你应该怎么理解

- 普通请求兼容：这是服务端 / bridge 的能力
- 多 agent 真能跑起来：这是客户端本地环境 + 服务端兼容共同决定的

不要把这两件事混成一件事。
